fix: Complete data analysis flow with database persistence

BREAKING CHANGES: None (backward compatible)

## Summary
Fixed critical issues in data analysis workflow from upload to results.
Flow now uses database persistence instead of in-memory cache.

## Changes

### Upload Flow (upload/route.ts)
- Add Supabase Storage upload for CSV files
- Create project in database (analysis_projects)
- Create variables in database (analysis_variables)
- Add authentication check
- Return real project ID from database

### Health Check Flow (health/route.ts)
- Remove error 400 response
- Load project from database
- Load CSV from Supabase Storage
- Execute health check on stored data
- Load variables from database

### Grouping Flow (group/route.ts)
- Remove in-memory cache
- Load project from database
- Load variables from database
- Generate suggestions from database data
- Remove unused imports (Papa, headers, preview)

### Page Logic (analysis/new/page.tsx)
- Remove separate health check API call
- Use health report from upload response
- Remove uploadedHeaders and uploadedPreview state
- Simplify handleHealthContinue
- Simplify handleHealthContinueAuto
- Simplify handleRefreshSuggestions

### New API Endpoint (variables/route.ts)
- Create GET /api/analysis/variables endpoint
- Load variables from database
- Convert database format to AnalysisVariable format

## Fixed Issues
1. ✅ Health check endpoint returning error 400
